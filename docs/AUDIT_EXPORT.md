# Audit Export Specification

> Jira: LNFTI-34

## Tujuan

Verifier dan admin dapat mengekspor data operasional LnFTI ke Excel atau CSV untuk audit, rekonsiliasi, pelaporan kampus, dan pemeriksaan insiden tanpa kembali menggunakan spreadsheet manual sebagai sumber utama data.

Jira dan PostgreSQL tetap menjadi sumber kebenaran. File ekspor hanya snapshot pada waktu tertentu.

## Hak Akses

- `student`: tidak dapat membuka halaman ekspor.
- `verifier`: dapat mengekspor data operasional yang tidak sensitif.
- `admin`: dapat menjalankan ekspor yang sama serta ekspor sensitif bila memiliki permission tambahan.

Setiap ekspor mencatat actor, waktu, format, filter, jumlah baris, status job, dan apakah data sensitif disertakan.

## Filter

- rentang tanggal;
- tipe laporan LOST/FOUND;
- report status;
- claim status;
- custody status;
- kategori;
- lokasi/gedung;
- verifier/actor;
- entity type untuk audit log.

Periode default adalah 30 hari. Ekspor tanpa batas tanggal tidak diperbolehkan melalui UI biasa.

## Workbook Excel

### Ringkasan

Berisi metadata ekspor, periode, filter, generated by, generated at, jumlah laporan, klaim, serah-terima, dan agregat status.

### Laporan

Kolom minimum:

```text
report_id
report_type
item_name
category
public_location
report_status
custody_status
event_at
created_at
published_at
resolved_at
reporter_reference
```

`reporter_reference` memakai identifier pseudonymized, bukan email atau nama lengkap.

### Klaim

```text
claim_id
report_id
claim_status
submitted_at
decided_at
verifier_reference
completed_at
```

Bukti kepemilikan tidak disertakan secara default.

### Serah Terima

```text
handover_id
report_id
claim_id
handover_at
handover_location
verifier_reference
created_at
```

### Audit Log

```text
timestamp
actor_reference
action
entity_type
entity_id
previous_status
new_status
safe_metadata
```

## CSV

CSV digunakan untuk interoperabilitas. Satu permintaan CSV mengekspor satu dataset yang dipilih: reports, claims, handovers, atau audit_logs. Encoding UTF-8 dengan BOM dapat digunakan agar mudah dibuka pada Excel versi Windows.

## Keamanan dan Privasi

- Ciri privat, bukti kepemilikan, kontak, email, dan OCR mentah dikecualikan secara default.
- Ekspor sensitif memerlukan permission tambahan dan alasan tertulis.
- Sel yang berasal dari input pengguna dan diawali `=`, `+`, `-`, atau `@` harus diperlakukan sebagai teks untuk mencegah spreadsheet formula injection.
- File disimpan sementara dan diunduh melalui signed URL dengan masa berlaku singkat.
- Nama file tidak mengandung nama mahasiswa atau informasi rahasia.
- Aktivitas pembuatan, keberhasilan, kegagalan, dan download dicatat pada audit log.

## Proses Job

```text
POST /api/admin/exports
→ validasi role dan filter
→ buat export_jobs:PENDING
→ worker mengambil snapshot konsisten
→ generate file
→ simpan sementara
→ export_jobs:COMPLETED atau FAILED
→ signed download URL tersedia
```

Untuk dataset kecil, implementasi awal boleh sinkron selama memiliki batas jumlah baris dan timeout aman. Dataset besar harus berpindah ke background job.

## Status Export Job

```text
PENDING
PROCESSING
COMPLETED
FAILED
EXPIRED
```

## Acceptance Criteria

- Hanya verifier/admin dapat mengekspor.
- Excel memiliki sheet Ringkasan, Laporan, Klaim, Serah Terima, dan Audit Log.
- CSV tersedia per dataset.
- Frozen header dan autofilter aktif pada sheet tabular.
- Format tanggal konsisten dengan timezone Asia/Jakarta.
- Data sensitif tidak ikut secara default.
- Formula injection dinetralisasi.
- Riwayat ekspor tampil pada dashboard.
- Download menggunakan URL sementara.
- Semua aktivitas ekspor diaudit.
